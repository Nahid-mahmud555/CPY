## 🚀 Overview

In an era where our daily workflows involve switching between heavy documentation, dense research papers, and continuous interactions with AI chat interfaces, productivity is frequently bottlenecked by a surprisingly mundane task: **copy-pasting and formatting text**. 

Manual text extraction is tedious, and traditional web clippers often introduce massive friction by forcing users to create accounts, route data through external cloud servers, or compromise their privacy. To eliminate these workflow inefficiencies, **CPY** was engineered to provide a lightning-fast, zero-friction text extraction and export mechanism directly inside the browser environment.

## 💡 Why CPY Was Built (The Core Need)

Modern power users, developers, and researchers handle vast amounts of unstructured digital text daily. Whether capturing code blocks from technical documentation or saving valuable analytical insights from AI conversations, the existing ecosystem of data-capturing tools suffers from critical shortcomings:

* **Cloud Dependency:** Many clippers upload captured content to third-party databases, raising privacy and security concerns.
* **Bloated Interfaces:** Users are often forced to navigate heavy dashboards or multi-step menus just to save a few lines of text.
* **Formatting Inefficiencies:** Raw text extraction frequently destroys structural hierarchies, making it difficult to utilize the data immediately in development or research pipelines.

CPY addresses these gaps by shifting the paradigm toward an **offline-first, single-click architecture** that prioritizes immediate utility and absolute user privacy.

## ⚡ Core Features & Benefits

CPY functions as a lightweight browser extension optimized for speed, precision, and multi-format versatility. 

* **One-Click Execution:** Users can capture targeted webpage text instantly without manually highlighting blocks or wrestling with complex menus.
* **Instant Multi-Format Export:** Extracted data can be exported on the fly into standard developer and researcher-friendly formats, including `JSON`, `PDF`, `CSV`, and `TXT`, fitting seamlessly into existing workflows.
* **Absolute Privacy via Local Processing:** Operating 100% offline, CPY ensures that no external tracking occurs and zero data leaves the browser, guaranteeing total data sovereignty.
* **Cross-Browser Roadmap & Contributions:** Fully available on Firefox, with active development underway for the official **Chrome Web Store** release. Developers and open-source contributors interested in helping optimize the codebase or assisting with Chrome publication are strongly encouraged to join forces.

# ❌ Browser Limitations

These are browser security restrictions—not bugs in CPY.

| Location | Why |
|----------|-----|
| Browser PDF Viewer (`file:///...pdf`) | Built-in PDF viewers are protected browser pages that extensions cannot access. |
| `chrome://`, `about:`, `edge://`, `view-source:` | Internal browser pages do not allow extensions to run. |
| Google Docs, Sheets & Slides | These apps render content using `<canvas>` instead of normal selectable HTML text. |

### PDF Workaround

Open the PDF using any web-based PDF viewer instead of the browser's built-in viewer.

Once the PDF loads as a normal webpage, CPY can capture text normally.
